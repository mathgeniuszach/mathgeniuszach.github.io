Origin Creator

This is a tool for creating custom datapacks for Minecraft in a more visual manner. It also mainly supports the Origins mod.

There may be bugs and performance issues (second-long loading times).

Join my Discord if you want to talk about the tool or some of my other projects.

Usage

You can click the "Help" button in the top left to return to this page at any time.

Before editing anything, you should add metadata to your work so people know it was you who made it. You can do this by clicking the "meta" item in the left box. This help page will then be replaced by an editor for metadata, and most of the entries in the editor are self-explanitory. If you don't understand something, you can hover over the label to the left of the entry box to view a quick description of that field. Blue labels you can also click on.

If you already have a datapack or mod you want to edit, press the "Import" button in the bottom left and select the item you want to edit on your Computer. You can also press the "Merge" button if you want to merge any datapack with your own without modifying metadata. Note that merging any datapack into your workspace will overwrite any files already being used (so be careful about the "minecraft:tick" and "minecraft:load" tags, or the "origins:origin" layer).

After adding metadata, you can add anything to the datapack by right clicking on any group or folder in the left pane and clicking "New File" (or if you want to make a folder, "New Folder"). Items may only contain letters, numbers, and underscores in their name.

Item names can optionally specify a namespace and/or a file extension in the format "namespace:name.ext". If a namespace is not found, it will default to whatever your pack's id is (e.g., "thing.json" becomes "mypack:thing.json"). If the file extention is not found on export, it will default to .json or .mcfunction (e.g., "thing" becomes "mypack:thing.json" or "mypack:thing.mcfunction"). Folders are considered too (e.g., "pack:thing.json" in the "folder" folder becomes "pack:folder/thing.json").

After adding anything, click on it to open up an editor for that item; you can switch between editors of different items by clicking them in the left pane. Right clicking any item in the left pane provides you the option to delete, cut, copy, and paste it anywhere. You can also pick up any added items and drag them around in the left pane to move them.

For information, here's what each of the different groups are:

  • Origin Layers
    Layers are groups of origins. Usually you will only need to use the origins:origin layer to enable people to select your origin when they first join a world. You can also make your own layers if you want to do additional things like conditional origins.
  • Origins
    Groups of powers that people can have and/or choose from, like the Blazeborn or the Enderian on the Origins homepage.
  • Powers
    Abilities and Disabilities that origins can have. They are very versitile.
  • Tags
    Lists of things. Note that any tag you create must be located in one of the predefined folders so minecraft knows the type of tag it is.
  • Functions
    Files containing a sequence of minecraft commands to run in order.
  • Predicates
    Data-driven conditions that the /execute if command can check for.
  • Recipes
    Custom crafting recipes for a variety of crafting blocks. Because the format is complex, the editor isn't as visual. Check out this tool to create a recipe in a more visual way. You can load individual recipes simply by pressing the "Toggle Raw" button at the top of an already created recipe, pasting the json code in there, and pressing "Save Raw".
  • Loot Tables
    Data structures storing how everything from how dungeon chests are generated to how mobs drop loot.
  • Advancements
    Goals the player can attempt to reach. Since most of the format is a bit complex, this editor isn't as visual. Check out this tool to create a bunch of advancements in a more visual way. You can merge any exported datapack (instead of importing it) to include them in your pack.
  • Data Scripts
    An unfinished programming language for creating mcfunctions and other files easier. This is just a teaser for now, and I don't know when it will be ready!

When you are finished editing, you can press the "To Datapack" button to export your work to a datapack or the "To Mod" button to export it to a mod. Note that empty folders will not be exported.

Tips

Creating a good origin can be difficult, so here's a few tips.

  • If you aren't sure what something does, hover over the label. A tooltip will provide a short description of what that entry is for.
  • This page auto-saves on your computer everytime a field loses focus, so refreshing or closing the page won't lose your progress. There's a save button in the top left too if you feel unsure. If you want to reset the pack back do it's default state, press "Reset".
  • Pressing the "+" button in any subpanel will add content to that panel and change the button to a "-" button. Pressing this new button will remove any content (and data stored) in that panel.
  • Want to save multiple projects? Click the "Raw Data" button, type anything into the "Save" box, and press the "Load" button to the bottom right. The page will be reloaded and you will store data in a separate location. The default save location is available if you put nothing in the save box and press "Load".
  • Check out the official wiki for more information on what each thing in the Origins mod does.
  • Check out the #origin-ideas chat in the official Origins Discord.
  • Test things out! If you aren't sure something will work, test it out in Minecraft! Failure is an important step in creativity.
  • Here are some good design principles to make origins/powers interesting:
    • Simple
      A good origin is easy to understand and doesn't have too many powers. Staying under six is a good idea and helps you make the most of each power. You can get around this by grouping powers together (e.g., if you can't touch water, this includes both water and rain).
    • Unique
      An origin should provide a gameplay experience and look and feel unique to itself. Copying a different origin in a slightly different way can get dull and old. This doesn't mean you can't take inspiration from places like mobs or other people's origins, it just means that people should be able to get a special experience using your origin.
    • Inspiring
      A good origin enables you to be creative with how you play the game, even if it's in a small way. Take for example the Merling; you must play underwater for most of the game, which requires the player to concoct new and interesting ways of doing things that would've been simple without the origin (e.g., how do you survive in the Nether?). You don't have to have a high-impact origin to do this (consider the Avian)
    • Balanced
      A good origin provides a good balance of power and disabilities. Making something too difficult (like having only half a heart) is only fun for a very small amount of people, and giving an origin too much power can take away the challenge that people also enjoy.
    • Fun
      A good origin provides a fun/challenging experience to a large subset of players. The definition of "Fun" depends on the person, but normally you can use common sense
